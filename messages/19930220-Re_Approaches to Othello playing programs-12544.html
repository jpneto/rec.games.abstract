Hmm.  Sorry but that's wrong.  The average number of moves available to a
player at any stage of the game is roughly 13.  Therefore the branching
factor is about 3.46 = square root of 13.


Now you have me confused!  Why do you take the square root of the number
of moves available?  Isn't the average branching factor just the average
number of moves available?  That's how much the game tree "branches" from
a node to its children.


Probably because the average effect of the alpha-beta heuristic is to reduce
the effective branching factor to approximately the square root of the real
branching factor. Since this is a big win, it's almost always a good idea to
use the alpha-beta heuristic, with the effect that the effective branching
factor is approximately the square root of the average number of moves
available.

So you're right: the real branching factor is just the average number of
moves available. It's just that the effective branching factor using
alpha-beta gives a better estimate of how deep you'll be able to search the
game tree.
