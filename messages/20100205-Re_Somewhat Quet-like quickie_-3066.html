There is a six-cell selection which fits on the
5x5 board but actually "covers" the 6x6 board:



+---+---+---+---+---+---+
| X |   | X |   |   |   |
+---+---+---+---+---+---+
|   |   |   |   |   |   |
+---+---+---+---+---+---+
| X |   |   |   | X |   |
+---+---+---+---+---+---+
|   |   |   |   |   |   |
+---+---+---+---+---+---+
|   |   | X |   | X |   |
+---+---+---+---+---+---+
|   |   |   |   |   |   |
+---+---+---+---+---+---+



Thus the minimum for size 6 can be lowered
to 6.  I wrote a Prolog program to solve
such puzzles, and it finds the solution
above in about 10 minutes.  I'm making
some tweaks to how it intializes the
board for speed, because you have to
search in some fashion the entire space
to confirm no smaller solution exists.

Hi, Phil:

I confirmed the minimum solutions for 7x7 and 8x8
boards both required 8 marked cells.  The program
finds solutions quickly, but exhaustively checking
nonexistence of 7 cell solutions takes 35 minutes
for the 7x7 board and 7 1/2 hours for the 8x8 board.

There's more efficiency I should implement, avoiding
checking symmetric configurations more than once.  I
place the "first mark" in roughly one-eighth of the
board, which exploits symmetry partially (as rotating
and reflecting the board can always thus position one
of the marked cells).  Fully exploiting the symmetry
of a square board is a topic that I remember kicking
around with someone (probably for N-queens) in another
