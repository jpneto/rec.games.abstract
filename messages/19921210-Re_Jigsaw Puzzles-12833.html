In article <1992Dec4.220321.18150@husc15.harvard.edu> blom@husc15.harvard.edu writes:

Does the complexity of solving a jigsaw puzzle increase on the order of the
square of the number of pieces in the puzzle?  Obviously if we compared each
piece with all the other pieces, this would be so. 
[...]  But the time series of the model might
still go up as O(n^2).  Is there any way to do say an FFT on something like
this?  Or is there some sort of parallel method of solution which would be
faster?


    If you split the puzzle pieces up into groups, then you would only 
have to fit the pieces within each group. Some nice grouping would be
edges, colors, patterns, shape... As you get more creative in grouping
pieces of the puzzle, the time it takes also go down. In fact, you can have 
levels of groupings, say first level by shape, and in each sub group by
color and so on. If you can divide down into sub groups of a few pieces,
your puzzle is only O(n*logm), where m is probalbly fractional unless you
subdivide by same number of catagorys every time.
