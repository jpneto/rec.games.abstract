The problem with "differentness" is that it relates the moves to their
ultimate goals, and relies on an analysis which is pretty
sophisticated, basically mapping the problem into an entirely
different but more effective representation.  This is certainly a key
area of research, but (at least in the general formulation) is not
useful as a simple metric on a game which could be automatically
detected.  

By the way, Dan, much of the work on these kind of
representation transformations in games has been done at UNC, by
Benerji, Ernst, and Goldstein.  Are you or anyone else on this group
involved in that work also?

Of course rule complexity is not guaranteed to be useful, just like
state complexity might not be useful.  I have received a similar
response over email from several people, who generally like you give
some modification of trivial go with some very complicated special
cases which don't seem to affect the game strategy much if at all.  
One point make against this is that while they only constrain some
*special case* which seems not to happen very often, it is possible
that this becomes the basis for a larger change in strategy.   
For example, consider the contrived rule you sketched above, which
gives a rule for one square on which you could not move based on my
last move.  With some calculation, I may be able to come up with a
strategy for the second player which made the last move of the first
player (which would normally win the game by placing the 181st stone) 
become unplayable.  Then the first player would have to pass, and the
second could play that square, thus winning the game!!!  
Thus what looks like a rare special case may give rise to deep
strategy.  

If we look at other known games with special-case rules (like
en-passant in chess), this is confirmed.  It is rare that you actually
take my pawn en-passant, the presence of this rule constrains many
variations in the openings and even in endgames.  Same with the rule
about not castling through check.  Even more special-case, look at the
k-move-rule for drawing a game.  The endings which are drawn because
of this are somewhat obscure, but knowing this fact about the endgame
often dictates which endgames we wish to enter, which has implications
right back into the opening.  

|> Some metric that short circuits this type of situation must be employed to
|> generate useful information about the relative complexity of different games.
|> 

I agree this must be done for distinctions which really are
disconnected from the goals of the game, but determining this requires
more analysis than any simple metric can provide.  

Perhaps the only conclusion we can reach on game complexity metrics is
that they can be used as guidelines by humans with respect to
particular games to argue about their relative complexity.  Still, I
would really like to have some good computable metrics, at least to use
as guidelines for an automatic game generator!
