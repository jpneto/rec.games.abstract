As Nick mentions, allowing backwards movement will greatly increase the
size of the game tree, as it will roughly double the number of squares
each piece can reach.  As far as I can tell, this will allow all
possible distributions of pieces on (and off) the board to be reached in
a game, so it would maximize the number of nodes in the game tree.

I can't see it as a complication of the rules, and it adds a strategic
choice: Do I move further away from my goal to block an enemy piece, or
do I move forward to get closer to my own goal?

If you also want to increase the number of edges, you can allow either
simultaneous move by two or more pieces, allow a piece to move more than
one square away, or allow a stack of pieces to move together.

Regardless, the size of the game tree is not necessarily a measure of
the quality of the game.  Sure, a very small tree will allow full
analysis, but even moderately-sized trees defeat this.  Also, there are
games with large game trees that are easily analysable (Nim, for
example), so to prove a game hard to analyse, you need to prove that no
"local" analysis can provide perfect play.  This has been proven for
many traditional games (by proving, e.g., that perfect play is in
EXP-time).  And even hard-to-analyse games are not necessarily
interesting: If the game is so hard to analyse that you have no
realistic way to judge if one move is better than another, playing is no
fun.
