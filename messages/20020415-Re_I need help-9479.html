What solution do you have? It seems that we get the P-positions
by starting with P_0 = {0, 0} and adding recursively for k >= 1

   P_k = {smallest positive integer not used, same + k}

This gives: {1, 2}, {3, 5}, {4, 7}, {6, 10}, {8, 13}, {9, 15},
{11, 18}, {12, 20}, {14, 23}, {16, 26}, {17, 28}, ...

I guess it's tricky to find a closed form expression for these.


That is what I got assuming the normal ending condition (you lose if
you can't move). I think the misre game will have a similar rule, but
the starting point will be {0, 1} instead of {1,2}. So that if you subtract
1 from both coordinates in a P position for the normal game, you get a P
position in the misre game.

It looks like the number P positions with both coordinates <= n is O(n).
(I think it is at least [n/4].) So if the original poster wants all of
the P positions for the 'solution', then it will take O(n) time to return
them.
