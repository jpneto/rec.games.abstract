 here is a rewording of some of what i posted here a while ago:

 Xong[tm] is an abstract game for 2 players.  it falls into the category
 of territory games. Xong is a position game, but there is no capturing.
 Xong never gets into a repetitive impasse or deadlock.  therefore it does
 not need any special rule to deal with that.  Xong is a strong candidate
 for solving a basic abstract game problem: the advantage of the first move.

 i have a website with some java applets (j2sdk1.4.0_01 applets) for single
 workstation play.  there are new-window browser links to the 7 rules, an

 pieces, a Standard game playing java applet and a Random game playing java
